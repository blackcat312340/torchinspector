# Requirements: TorchInspector (Archived v1.2)

**Defined:** 2026-06-08
**Archived:** 2026-06-15
**Core Value:** Make the internal state of PyTorch training loops observable through a clean, minimal API.

## v1 Requirements — Complete

All v1 requirements validated across Phases 1-6.

### Core Observation (CORE)

- [x] **CORE-01**: Inspector wraps nn.Module + optimizer with auto step counting and hook lifecycle — Phase 1
- [x] **CORE-02**: Scalar metrics logging (loss, accuracy, lr, batch/epoch time, GPU memory) — Phase 1
- [x] **CORE-03**: Parameter weight histograms per layer at configurable intervals — Phase 1
- [x] **CORE-04**: Parameter gradient histograms per layer at configurable intervals — Phase 1
- [x] **CORE-05**: Model computation graph logging via `inspector.log_graph(dummy_input)` — Phase 1
- [x] **CORE-06**: ONNX export for Netron visualization — Phase 1

### Layer Monitoring (WATCH)

- [x] **WATCH-01**: Specify layers by name for monitoring — Phase 1
- [x] **WATCH-02**: Wildcard pattern matching for layer selection — Phase 2
- [x] **WATCH-03**: Forward hooks with overwrite pattern (no memory leak) — Phase 1
- [x] **WATCH-04**: Activation statistics (mean, std, min, max) at intervals — Phase 2
- [x] **WATCH-05**: Activation sparsity ratio for dead ReLU detection — Phase 2
- [x] **WATCH-06**: Gradient norm per watched layer at intervals — Phase 2
