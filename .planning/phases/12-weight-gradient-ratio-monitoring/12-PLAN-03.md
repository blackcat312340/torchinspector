# Phase 12 Plan 03: Inspector Integration + TensorBoard Output

**Requirements:** INT-01 (partial — unified TrendMonitor), INT-02 (partial — cross-metric correlation)
**Goal:** Wire `WeightGradRatioCollector` into the `Inspector` facade, integrate `check_wgr()` into the step loop, and verify end-to-end TensorBoard output.

---

## Context

Plans 01 and 02 created:
- `WeightGradRatioCollector` — core collector with backward hooks, log-space ratio, mean+max aggregation
- `TrendMonitor.check_wgr()` — multi-scale trend detection with escalation

This plan integrates both into the `Inspector` facade so users get WGR monitoring automatically when they call `inspector.step()`.

The integration follows the same pattern as `GradientCollector`:
- Constructor initializes the collector
- `step()` calls `collect()` at log interval
- `close()` cleans up hooks

## Task 2: Initialize `WeightGradRatioCollector` in `Inspector.__init__()`

**File:** `src/torchinspector/inspector.py`

### Add import:

```python
from torchinspector.collectors.weight_grad_ratio import WeightGradRatioCollector
```

### Add initialization (after `_gradient_collector`):

```python
self._weight_grad_ratio_collector = WeightGradRatioCollector(
    model,
    self._hook_manager,
    self._backend,
    self._monitor,
    log_interval=log_interval,

```

**Note:** The collector takes `monitor` as a parameter so it can call `check_wgr()` during collection. This differs from `GradientCollector` which doesn't interact with the monitor directly.

## Task 5: Clean up hooks in `Inspector.close()`

**File:** `src/torchinspector/inspector.py`

Add cleanup call before `self._hook_manager.remove_all()`:

```python
def close(self) -> None:
    if self._closed:
        return
    self._weight_grad_ratio_collector.close()  # Remove backward hooks
    self._hook_manager.remove_all()
    self._backend.close()
    self._closed = True
```

**Why before hook_manager:** The collector's backward hooks are independent of HookManager's forward hooks. Cleaning up the collector first ensures its hooks are removed before the model state changes.
