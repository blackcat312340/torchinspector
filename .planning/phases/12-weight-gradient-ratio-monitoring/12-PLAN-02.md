# Phase 12 Plan 02: TrendMonitor Integration + Alert

**Requirements:** WGR-02 (vanishing/exploding detection + TrendMonitor alert), WGR-04 (multi-scale windows 10/50/200)
**Goal:** Add `check_wgr()` method to TrendMonitor with multi-scale trend detection and alert escalation for W/G ratio monitoring. Add correlation rules linking WGR anomalies to convergence and gradient signals.

---

## Context

The existing `TrendMonitor` (`src/torchinspector/monitor.py`) already supports:
- Multi-scale sub-windows via `check_convergence()` (short=10, medium=50, long=200)
- Slope computation via `_compute_slope()`
- Alert escalation (OK → INFO → WARN → CRITICAL)
- Correlation rules in `correlation_check()`

Phase 12 adds a new `check_wgr()` method that reuses the same multi-scale window pattern but with WGR-specific trend detection logic: instead of checking a metric against a fixed threshold, it checks whether the log-space ratio is *consistently rising* (vanishing signal) or *consistently falling* (exploding signal) across multiple time scales.

## Task 1: Add `check_wgr()` method to TrendMonitor

**File:** `src/torchinspector/monitor.py`

### Method signature:

```python
def check_wgr(self, name: str, log_ratio: float, step: int) -> AlertLevel:
```

### Implementation:

1. **Feed three sub-windows** (same pattern as `check_convergence()`):
   ```python
   for suffix, size in [
       (":short", _SHORT_WINDOW),    # 10 steps
       (":medium", _MEDIUM_WINDOW),  # 50 steps
       (":long", _LONG_WINDOW),      # 200 steps
   ]:
       key = f"ratios/{name}/mean{suffix}"
       win = self._windows[key]
       win.append(log_ratio)
       if len(win) > size:
           win.pop(0)
   ```

2. **Compute slopes** for short and long windows:
   ```python
   short_key = f"ratios/{name}/mean:short"
   long_key = f"ratios/{name}/mean:long"
   short_slope = self._compute_slope(self._windows.get(short_key, []))
   long_slope = self._compute_slope(self._windows.get(long_key, []))
   ```

3. **Trend detection logic:**
   - `alert_key = f"wgr/{name}"`
   - If both slopes positive → vanishing trend (weight dominating gradient) → increment `_alert_counts[alert_key]`
   - If both slopes negative → exploding trend (gradient dominating weight) → increment `_alert_counts[alert_key]`
   - If slopes disagree → mixed signal → decay count by 1
   - If both negative and improving → reset to OK

4. **Escalation thresholds:**
   - `count >= 5` → `AlertLevel.INFO`
   - `count >= 10` → `AlertLevel.WARN`
   - `count >= 20` AND acceleration (`abs(short_slope) > abs(long_slope) * 1.5`) → `AlertLevel.CRITICAL`

5. **Store current alert level:**
   ```python
   self._current_alerts[alert_key] = level
   ```

## Task 2: Update `correlation_check()` with WGR-specific rules

**File:** `src/torchinspector/monitor.py`

### Rule 1: `wgr_abnormal AND convergence_slow → CRITICAL`

```python
# In correlation_check():
wgr_keys = [k for k in metrics if "ratios/" in k and ":short" not in k and ":medium" not in k and ":long" not in k]
if self._last_convergence_score is not None and self._last_convergence_score < 40:
    for k in wgr_keys:
        win = self._windows.get(k, [])
        if win:
            latest = win[-1]
            if latest > 6.0 or latest < -6.0:  # log_ratio thresholds
                alerts.append((
                    "convergence_slow_wgr_abnormal",
                    AlertLevel.CRITICAL,
                    "Slow convergence + abnormal W/G ratio — possible vanishing/exploding gradient",
                ))
                break
```

### Rule 2: `wgr_vanishing AND gradient_declining → WARN`

```python
grad_keys_filtered = [k for k in metrics if "gradient" in k and "norm" in k]
for k in wgr_keys:
    wgr_win = self._windows.get(k, [])
    wgr_slope = self._compute_slope(wgr_win)
    if wgr_slope is not None and wgr_slope > 0:  # Rising = vanishing trend
        for gk in grad_keys_filtered:
            g_slope = self._compute_slope(self._windows.get(gk, []))
            if g_slope is not None and g_slope < 0:
                alerts.append((
                    "wgr_vanishing_gradient_declining",
                    AlertLevel.WARN,
                    "W/G ratio rising + gradients falling — vanishing gradient confirmed",
                ))
                break
```

**Note:** The existing `correlation_check()` already has a `convergence_slow_wgr_abnormal` rule (lines 221-234) that uses raw ratio thresholds (1000/0.001). We need to **update** this rule to use log-space thresholds (6.0/-6.0) for consistency with the new collector's output format.

wgr_alert_keys = [k for k in self._current_alerts if k.startswith("wgr/")]
if wgr_alert_keys:
    ok_count = sum(1 for k in wgr_alert_keys if self._current_alerts[k] == AlertLevel.OK)
    warn_count = sum(1 for k in wgr_alert_keys if self._current_alerts[k] >= AlertLevel.WARN)
